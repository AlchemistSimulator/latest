Classes

Name Description
open class Continuous2DEnvironment<T> : Abstract2DEnvironment<T, Euclidean2DPosition>, Physics2DEnvironment<T>

Implementation of Physics2DEnvironment.
class ImageEnvironmentWithGraph<T> : ImageEnvironment<T>, EuclideanPhysics2DEnvironmentWithGraph<RectObstacle2D<Euclidean2DPosition>, T, ConvexPolygon, Euclidean2DPassage>

An ImageEnvironment providing an Euclidean2DNavigationGraph. The NaviGator algorithm is used to produce such graph (see generateNavigationGraph). The positions where to plant initial seeds should be specified directly in the image, marking each area of the environment with one or more pixels of a given color (defaults to blue).
class Rectangular2DEnvironment<T> : Continuous2DEnvironment<T>, HasBoundaries

A bounded Physics2DEnvironment, nodes cannot exist nor move outside the rectangular boundaries centered in (0,0).
open class Continuous2DObstacles<T : Any?> : LimitedContinuos2D<T>, EuclideanPhysics2DEnvironmentWithObstacles<RectObstacle2D<Euclidean2DPosition?>?, T>
class InfiniteHalls<T : Any?> : LimitedContinuos2D<T>
class MuseumHall<T : Any?> : Continuous2DEnvironment<T>
abstract class AbstractEnvironment<T : Any?, P : Position<P>?> : Environment<T, P>, Any

Very generic and basic implementation for an environment. Basically, only manages an internal set of nodes and their position.
abstract class Abstract2DEnvironment<T : Any?, P : Position2D<P>?> : AbstractEnvironment<T, P>

Models a bidimensional environment.
open class ImageEnvironment<T : Any?> : Continuous2DObstacles<T>

This environment loads an image from the file system, and marks as obstacles all the pixels of a given color.
abstract class LimitedContinuos2D<T : Any?> : Continuous2DEnvironment<T>

This class represents a 2D continuous environment with spatial limitations. Those limitations will prevent nodes to move in positions which are not allowed.
class BioRect2DEnvironmentNoOverlap: BioRect2DEnvironment, EnvironmentSupportingDeformableCells<Euclidean2DPosition?>

Implements a limited environment supporting cells with a defined shape, avoiding any overlapping among them.
open class BioRect2DEnvironment: LimitedContinuos2D<Double?>
class OSMEnvironment<T : Any?> : Abstract2DEnvironment<T, GeoPosition?>, MapEnvironment<T>

This class serves as template for more specific implementations of environments using a map. It encloses the navigation logic, but leaves the subclasses to decide how to provide map data (e.g. loading from disk or rely on online services). The data is then stored in-memory for performance reasons.
