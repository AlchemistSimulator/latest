getNeighborPropensity

protected abstract fun getNeighborPropensity(neighbor: Node<T>?): Double

Given a node, which is supposed to be in the neighborhood of the current node, the function computes a double value representing the propensity of the neighbor to be the chosen one for the reaction to be executed. The value returned must be 0 if the neighbor is not eligible for the reaction due to this condition. This value could be used to compute the reaction's propensity, but the main usage is to give a rate to every neighbor and randomly choose one of them.

fun getValidNeighbors(): MutableMap<Node<T>?, Double?>?

Searches in the given neighborhood which nodes satisfy the condition, and returns a list of valid neighbors. NOTE, it is NOT guaranteed that this method checks if the passed neighborhood is the actual neighborhood of the node. Make sure the passed neighborhood is up to date for avoid problems.
